## üîç Key Design Principles for V6

### **Planning Input Handling**
- **Assumption:** Planning artifacts are complete and locked by Phase 8
- **Focus:** Validate consistency, not completeness
- **Constraint:** Cannot modify planning artifacts - must make them pass
- **Interface:** Loose coupling via clear artifact format specifications

### **Fresh Instance Trigger Precision**
Based on meta-process architecture requirements:

**Trigger Fresh Instance:**
- Test specification contradicts test implementation
- Pseudocode assumptions prove invalid during implementation
- Architecture constraints cannot be satisfied
- External dependency specifications are incorrect

**Do NOT Trigger Fresh Instance:**
- Import errors (SimpleFailure)
- Logic bugs in implementation
- Missing implementation methods
- Performance optimization needs

### **Evidence Package Content**
**Required Components:**
- Specific planning artifact section with inconsistency
- Implementation context showing the contradiction
- Concrete examples of the specification failure
- Proposed resolution or clarification needed

**Format:** Structured for fresh instance evaluation without implementation bias

### **Meta-Process Integration**
- **Micro-iterations:** Within implementation (V5-style decisions)
- **Macro-iterations:** Cross-phase returns (back to earlier planning phases)
- **Meta-iterations:** Cross-process returns (challenge planning artifacts via fresh instance)
