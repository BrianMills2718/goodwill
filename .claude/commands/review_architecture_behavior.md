# /review_architecture_behavior

Review architecture and behavior documentation for strategic guidance.

## What This Command Does

1. Checks architecture documentation for design patterns
2. Reviews behavior requirements for clarity
3. Identifies alignment with project goals
4. Extracts guidance for uncertainties

## When to Use

- When categorized as strategic uncertainty
- When design decisions needed
- When requirements ambiguous
- For architectural guidance

## Actions

1. Read `docs/architecture/technical_design.md`
2. Read `docs/behavior/requirements.md`
3. Check phase-specific requirements
4. Extract relevant guidance
5. Document findings
6. Update understanding

## Evidence Collection

Create: `investigations/strategic/architecture_review.md`
- Questions addressed
- Guidance found
- Decisions made
- Remaining gaps

## Next Command

- **Guidance found** → Return to `/implement` or `/write_tests`
- **Still unclear** → `/investigate_pre_impl`
- **Can't resolve** → `/escalate_to_deferred`

## Success Criteria

- Strategic questions answered
- Clear architectural direction
- Requirements understood
- Ready to proceed
