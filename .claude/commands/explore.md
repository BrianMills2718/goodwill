# /explore

Explore the codebase and understand the requirements for the current phase.

## What This Command Does

1. Reads relevant documentation (requirements, architecture, phases)
2. Examines existing code structure
3. Identifies uncertainties and unknowns
4. Documents findings in investigations/
5. Prepares for test writing

## When to Use

- After loading a phase
- When starting a new feature
- After resolving blockers
- When context is unclear

## Actions

1. Read docs/behavior/requirements.md
2. Read docs/architecture/technical_design.md
3. Check current phase requirements
4. Explore existing codebase structure
5. Identify what needs to be built
6. Document uncertainties in investigations/
7. Create initial understanding document

## Next Command

After exploration, proceed to `/write_tests` to define success criteria using TDD approach.
